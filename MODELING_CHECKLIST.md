# 建模回归检查清单

本清单用于字段建模能力的回归验证。

适用场景：
- 新增字段类型
- 修改字段协议
- 调整模型配置弹窗
- 修改 `schema` / `runtime` / `crud` 云函数
- 发布前自检

## 1. 通用检查

1. 打开模型创建页，确认字段类型分组、命名、说明正常。
2. 新增字段弹窗中，字段类型切换后，相关配置项能正确出现或隐藏。
3. 新增字段后，字段列表中的摘要、标签、类型名称显示正确。
4. 编辑已有字段时，所有配置都能正确回填。
5. 保存模型后重新进入编辑页，配置不丢失、不串字段。
6. 删除字段、调整字段顺序后，保存与重新进入编辑页仍然正确。

## 2. 基础字段

### 2.1 文本

覆盖字段：
- `text`
- `textarea`
- `richtext`
- `markdown`

检查项：
1. 默认值可保存并正确回填。
2. `minLength` / `maxLength` 可保存并正确回填。
3. 最小长度大于最大长度时，前端会拦截。
4. 运行时录入时，长度约束生效。
5. 列表、详情、编辑回显正常。

### 2.2 数字

覆盖字段：
- `number`

检查项：
1. 默认值可保存并正确回填。
2. `minValue` / `maxValue` 可保存并正确回填。
3. 最小值大于最大值时，前端会拦截。
4. 运行时录入非法数字时，前端会拦截。
5. `crud` 保存时数值校验生效。

### 2.3 布尔

覆盖字段：
- `boolean`

检查项：
1. 默认值可选择并回填。
2. 运行时可选择是 / 否。
3. 列表、详情展示正常。

### 2.4 日期时间

覆盖字段：
- `date`
- `datetime`

检查项：
1. 默认值可保存并回填。
2. `dateStorageFormat` 可保存并回填。
3. 非法日期值会被拦截。
4. 列表、详情、编辑页展示一致。

## 3. 结构字段

### 3.1 枚举

覆盖字段：
- `enum`

检查项：
1. 至少需要一个枚举选项。
2. 选项的 `label` / `value` 缺失时不能保存。
3. 默认值只能从已有选项中选择。
4. 编辑时枚举选项和默认值回填正确。
5. 运行时录入和列表展示正常。

### 3.2 数组

覆盖字段：
- `array`

检查项：
1. `itemType` 可配置并回填。
2. `minItems` / `maxItems` 可配置并回填。
3. 默认值必须是合法 JSON 数组。
4. 最少项数大于最多项数时，前端会拦截。
5. `crud` 保存时数组校验生效。

### 3.3 JSON

覆盖字段：
- `json`

检查项：
1. `jsonValueType` 可配置并回填：
   - `any`
   - `object`
   - `array`
2. 默认值格式必须和 `jsonValueType` 匹配。
3. 非法 JSON 无法保存。
4. 运行时录入与详情展示正常。

## 4. 地理字段

### 4.1 位置

覆盖字段：
- `location`

检查项：
1. `locationCoordinateSystem` 可保存并回填。
2. `locationRequireAddress` / `locationRequireName` 可保存并回填。
3. `locationStorageMode` 可在 `object` / `flat` 间切换。
4. `flat` 模式下映射字段必须填写完整。
5. 默认值可正确构建并回填。
6. 运行时录入与 `crud` 存取一致。

### 4.2 地址

覆盖字段：
- `address`

检查项：
1. `addressGranularity` 可保存并回填。
2. `addressStorageMode` 可在 `object` / `flat` 间切换。
3. `flat` 模式下映射字段必须填写完整。
4. 默认值可正确构建并回填。
5. 运行时录入与 `crud` 存取一致。

## 5. 资源字段

覆盖字段：
- `image`
- `file`
- `video`
- `audio`

检查项：
1. `accept` 可配置并回填。
2. `maxFileSizeMB` 可配置并回填。
3. 单值 / 多值模式切换正确。
4. `minItems` / `maxItems` 可配置并回填。
5. 上传成功后值可正确保存。
6. 列表、详情展示正常。
7. 非法文件类型或超限文件会被拦截。

## 6. 关联字段

### 6.1 单模型（单记录）

覆盖字段：
- `relation`

检查项：
1. 必须选择关联模型。
2. 至少配置一个展示字段。
3. 保存模型后重新进入编辑页，回填正确。
4. 运行时录入和展示正常。

### 6.2 单模型（多记录）

覆盖字段：
- `multiRelation`

检查项：
1. 必须选择关联模型。
2. 至少配置一个展示字段。
3. `minItems` / `maxItems` 可配置并回填。
4. 运行时录入少于最少值时报错。
5. 运行时录入超过最多值时报错。
6. 列表、详情、编辑回显正常。

### 6.3 多模型（单记录）

覆盖字段：
- `polyRelation`

检查项：
1. 至少选择一个允许关联的模型。
2. 每个模型都必须配置至少一个展示字段。
3. 保存模型后重新进入编辑页，模型列表和展示字段回填正常。
4. 运行时录入可按结构化方式新增：
   - 先选模型
   - 再填记录 ID
5. 非允许模型会被拦截。

### 6.4 多模型（多记录）

覆盖字段：
- `multiPolyRelation`

检查项：
1. 至少选择一个允许关联的模型。
2. 每个模型都必须配置至少一个展示字段。
3. 每个模型都可分别配置：
   - 最少关联数
   - 最多关联数
4. 保存模型后重新进入编辑页，以下内容回填正常：
   - `relationModelCollections`
   - `polyRelationDisplayMap`
   - `polyRelationLimitMap`
5. 运行时录入支持结构化新增：
   - 点击新增关联
   - 每条先选模型
   - 再填记录 ID
   - 支持删除
6. 顶部模型数量提示正常。
7. 同一 `collection + id` 重复时会被拦截。
8. 某个模型少于最少关联数时报错。
9. 某个模型超过最多关联数时报错。

## 7. 默认值规则

以下字段不应提供默认值输入：
- `relation`
- `multiRelation`
- `polyRelation`
- `multiPolyRelation`

检查项：
1. 模型配置弹窗中不出现默认值输入。
2. 编辑已有字段时，不会出现默认值残留。
3. `schema` 云函数保存时会强制清空这些字段的默认值。

## 8. 字段摘要检查

1. 字段列表摘要与字段实际配置一致。
2. 关联字段命名统一为：
   - `单模型（单记录）`
   - `单模型（多记录）`
   - `多模型（单记录）`
   - `多模型（多记录）`
3. `multiPolyRelation` 摘要中应展示按模型分别配置的关联数限制。

## 9. 协议一致性检查

以下文件中的字段协议必须保持一致：

前端主源：
- `web-admin/src/runtime/fieldTypes/metaSource.ts`

云函数部署副本：
- `cloudfunctions/schema/src/field-meta.js`
- `cloudfunctions/runtime/src/field-meta.js`
- `cloudfunctions/crud/src/field-meta.js`

检查项：
1. 字段 `label` 一致。
2. `formRenderer` / `displayRenderer` 一致。
3. `supports` 一致。
4. 新增字段类型后，四处都同步。

## 10. 发布前最低验证

1. `web-admin` 构建通过。
2. 至少手工验证以下流程：
   - 模型新建
   - 模型编辑回填
   - 运行时新增记录
   - 运行时编辑记录
   - 列表与详情展示
3. 若改动涉及云函数：
   - 检查 `schema`
   - 检查 `runtime`
   - 检查 `crud`
4. 若当前部署环境的云函数代码不直接来自本仓库：
   - 手动同步 `field-meta.js`
   - 手动同步对应 `index.js`

## 11. 当前重点风险点

若后续继续迭代，优先关注以下区域：
- 关联字段协议变更时，前端主源与三份云函数副本是否同步
- `multiPolyRelation` 的按模型限制是否仍然正确回填和校验
- `location` / `address` 的 `flat` 模式存取是否回归
- 资源字段的多值与大小限制是否回归
